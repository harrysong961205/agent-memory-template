기능을 추가/수정할 때 이 맵을 참고하여 영향받는 모든 곳을 확인한다.

> 범례: 프로젝트에 맞게 수정한다. 예) `WEB` = 웹, `APP` = 모바일, `SV` = 서버

## 변경 시 체크 규칙

### API 엔드포인트를 수정할 때
1. 서버 라우트/컨트롤러 수정
2. 타입 변경 반영
3. 이 맵에서 해당 기능의 **모든 포탈** 확인
4. 각 포탈의 API 호출 코드 업데이트

### 새 기능을 추가할 때
1. 이 맵에서 SHARED/플랫폼별 중 어디에 해당하는지 분류
2. SHARED라면 모든 포탈에 구현 계획 필요
3. **이 파일에 새 항목 추가**

### DB 스키마를 변경할 때
1. 마이그레이션 생성
2. `SCHEMA_SUMMARY.md` 갱신
3. 이 맵에서 해당 모델을 사용하는 **모든 포탈** 확인
